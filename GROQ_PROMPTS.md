# Groq AI Prompts for Comprehensive Company Data

## Prompt 1: Multi-Specialty Hospital (Real-Time Data)

```
Generate comprehensive, realistic data for a multi-specialty hospital in JSON format. Include ALL of the following details:

**Hospital Name:** Apollo Hospitals, Hyderabad
**Location:** Jubilee Hills, Hyderabad, Telangana, India

Please provide:

1. **Hospital Overview:**
   - Full name and tagline
   - Year established
   - Total bed capacity
   - Number of ICU beds
   - Emergency services (24/7 availability)
   - Accreditations (NABH, JCI, etc.)
   - Contact information (phone, email, website)
   - Address with pincode
   - Operating hours for different departments

2. **Departments (at least 15):**
   For each department include:
   - Department name
   - Head of department
   - Number of beds
   - Facilities available
   - Special equipment
   - Operating hours

3. **Doctors (at least 25):**
   For each doctor include:
   - Full name with title (Dr.)
   - Specialization
   - Department
   - Qualifications (MBBS, MD, DM, etc.)
   - Years of experience
   - Consultation fee
   - Available days (e.g., Mon, Wed, Fri)
   - Available time slots (e.g., 9:00 AM - 1:00 PM, 4:00 PM - 7:00 PM)
   - Languages spoken
   - Special interests/expertise
   - OPD room number

4. **Services Offered (at least 30):**
   - Diagnostic services (MRI, CT Scan, X-Ray, Ultrasound, etc.)
   - Laboratory tests
   - Surgical procedures
   - Therapeutic services
   - Preventive health checkups
   - Emergency services
   - Ambulance services
   - Pharmacy services
   - Blood bank
   - Physiotherapy
   - Dialysis
   - Maternity services
   - Pediatric care
   - Geriatric care

5. **Health Packages:**
   - Package name
   - Tests included
   - Price
   - Duration
   - Target age group

6. **Appointment Slots:**
   - Available time slots for next 7 days
   - Department-wise availability
   - Doctor-wise availability
   - Already booked slots (realistic sample data)

7. **Emergency Contact:**
   - Ambulance number
   - Emergency department number
   - Blood bank emergency number
   - Trauma center number

8. **Facilities:**
   - Parking (capacity, charges)
   - Cafeteria (timings, menu highlights)
   - ATM availability
   - Pharmacy (24/7 or timings)
   - Waiting areas
   - Patient rooms (types: general, semi-private, private, deluxe, suite with prices)

9. **Insurance & Payment:**
   - Accepted insurance providers (at least 15)
   - Payment modes accepted
   - Cashless facility availability

10. **Patient Feedback (Sample 10):**
    - Patient name (anonymized)
    - Department visited
    - Doctor consulted
    - Rating (1-5)
    - Review text
    - Date of visit

11. **Common Queries & Answers:**
    - How to book an appointment?
    - What documents are needed for admission?
    - Visiting hours for patients
    - COVID-19 protocols
    - How to get medical records?
    - Refund policy
    - Second opinion process

12. **Specializations & Treatments:**
    - List of complex surgeries performed
    - Success rates (where applicable)
    - International patient services
    - Telemedicine availability

Format the output as a structured JSON that can be directly used in the AI calling agent system. Make it as detailed and realistic as possible with Indian context (names, locations, pricing in INR, etc.).
```

## How to Use These Prompts:

1. **Copy the entire prompt** (including the triple backticks)
2. **Paste it into your Groq API chat** or any AI interface
3. **Get the JSON output**
4. **Parse and store** the data in your application's database
5. **Use it** to populate the company context in your AI calling agent

---

## Expected JSON Structure:

The AI should return data in this format:

```json
{
  "companyInfo": {
    "name": "Apollo Hospitals",
    "industry": "Healthcare",
    "established": 1983,
    // ... more fields
  },
  "departments": [
    {
      "name": "Cardiology",
      "head": "Dr. Rajesh Kumar",
      // ... more fields
    }
  ],
  "doctors": [
    {
      "name": "Dr. Priya Sharma",
      "specialization": "Neurology",
      // ... more fields
    }
  ],
  // ... more sections
}
```

You can then use this data to populate your `contextSummary` or `nlpContext` field when creating a company in your system.
